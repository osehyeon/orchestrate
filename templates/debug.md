# 디버깅 템플릿

## 사용법

Claude Code에서 아래 프롬프트의 `{{변수}}`를 실제 값으로 바꿔서 입력한다.

## 프롬프트

```
다음 버그를 조사하기 위한 에이전트 팀을 만들어줘.

**증상**: {{버그 증상 설명}}
**재현 조건**: {{재현 방법 — 알고 있다면}}
**의심되는 영역**: {{관련 파일/모듈 — 알고 있다면}}

팀 구성 — 각자 다른 가설을 세우고 검증한다:
1. researcher 에이전트 타입으로 "가설A" 팀원을 스폰해줘 — {{첫 번째 가설}} 방향으로 조사한다.
2. researcher 에이전트 타입으로 "가설B" 팀원을 스폰해줘 — {{두 번째 가설}} 방향으로 조사한다.
3. researcher 에이전트 타입으로 "가설C" 팀원을 스폰해줘 — {{세 번째 가설}} 방향으로 조사한다.

팀원들이 서로의 가설을 검증하고 반박하도록 해줘.
리드(당신)는 조사 결과를 종합하여 가장 유력한 원인을 판단한다.
원인이 확정되면 implementer 에이전트 타입으로 "수정자"를 추가로 스폰해서 수정한다.
판단 기준: 2명 이상의 조사자가 동일 원인을 지목하거나, 재현 가능한 근거가 제시된 경우.
```

## 태스크 흐름

```
[가설A: 조사 + 검증] ──┐
[가설B: 조사 + 검증] ──┼──→ [리드: 원인 확정] ──→ [수정자: 버그 수정]
[가설C: 조사 + 검증] ──┘
```

## 가설 설정 가이드

가설을 모를 때는 다음처럼 넓게 시작한다:
- 가설A: "입력 데이터 문제" — 입력 검증, 데이터 형식 관련 조사
- 가설B: "상태 관리 문제" — 경쟁 상태, 초기화 순서 관련 조사
- 가설C: "외부 의존성 문제" — API 응답, 라이브러리 버전 관련 조사
