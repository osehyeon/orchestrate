# 기술 조사/비교 분석 템플릿

## 사용법

Claude Code에서 아래 프롬프트의 `{{변수}}`를 실제 값으로 바꿔서 입력한다.

## 프롬프트

```
다음 주제를 조사하기 위한 에이전트 팀을 만들어줘.

**조사 주제**: {{조사할 기술/도구/접근법}}
**목적**: {{왜 조사하는가 — 기술 선택, 마이그레이션 결정 등}}
**비교 대상**: {{비교할 후보 목록}}
**평가 기준**: {{성능, 비용, 생태계, 학습 곡선 등}}

팀 구성:
1. researcher 에이전트 타입으로 "조사자A" 팀원을 스폰해줘 — {{후보 1}}을 심층 조사한다.
2. researcher 에이전트 타입으로 "조사자B" 팀원을 스폰해줘 — {{후보 2}}를 심층 조사한다.
3. researcher 에이전트 타입으로 "조사자C" 팀원을 스폰해줘 — {{후보 3}}을 심층 조사한다.
4. devils-advocate 에이전트 타입으로 "비평가" 팀원을 스폰해줘 — 각 조사자의 결론에 반론을 제기한다.

조사자들이 각자 조사를 마치면 리드(당신)가 각 조사 결과를 다른 조사자들에게 공유해줘.
각 조사자는 다른 후보의 조사 결과에 대해 메시지로 반론이나 보완 의견을 보낸다.
비평가는 각 후보의 약점을 집중적으로 파헤친다.
```

## 태스크 흐름

```
[조사자A: 후보1 조사] ──┐
[조사자B: 후보2 조사] ──┼──→ [리드가 결과 상호 배포] ──→ [각 조사자 반론 라운드] ──→ [비평가 검증] ──→ [리드: 비교표 + 권장안]
[조사자C: 후보3 조사] ──┘

※ 병렬: 초기 조사 3건은 서로 독립.
※ 직렬: 서브에이전트 간 직접 통신이 없으므로 리드가 명시적으로 조사 결과를 다른 조사자에게 전달해야 한다 (상호 배포 단계).
※ 비평가는 전체 반론 라운드가 끝난 후 시작.
```

## 결과물 구조 (권장)

경로는 CLAUDE.md의 "작업 디렉토리 규칙"을 따른다. `{작업디렉토리}`는 사용자가 지정한 폴더거나, 미지정 시 `output/{작업슬러그}/`이다.

```
{작업디렉토리}/
├── comparison.md           # 최종 비교 분석 보고서
├── candidates/
│   ├── candidate_a.md      # 후보 1 상세 분석
│   ├── candidate_b.md      # 후보 2 상세 분석
│   └── candidate_c.md      # 후보 3 상세 분석
└── recommendation.md       # 최종 권장안 + 근거
```
