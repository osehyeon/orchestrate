# 과제/제안서 작성 템플릿

## 사용법

Claude Code에서 아래 프롬프트의 `{{변수}}`를 실제 값으로 바꿔서 입력한다.

## 프롬프트

```
다음 제안서를 작성하기 위한 에이전트 팀을 만들어줘.

**제안서 주제**: {{주제}}
**제안 대상**: {{제출 대상 — 학교, 기업, 정부 기관 등}}
**핵심 요구사항**: {{제안서에 반드시 포함해야 할 내용}}
**분량/형식**: {{페이지 수, 형식 제한 등}}
**마감일**: {{날짜}}

팀 구성:
1. researcher 에이전트 타입으로 "배경조사" 팀원을 스폰해줘 — 주제 관련 현황, 선행 연구, 시장 분석을 조사한다.
2. researcher 에이전트 타입으로 "기술조사" 팀원을 스폰해줘 — 기술적 실현 가능성, 관련 기술/도구를 조사한다.
3. architect 에이전트 타입으로 "설계자" 팀원을 스폰해줘 — 제안할 시스템/솔루션의 아키텍처를 설계한다.
4. writer 에이전트 타입으로 "집필자" 팀원을 스폰해줘 — 조사 결과와 설계를 바탕으로 제안서를 작성한다.
5. devils-advocate 에이전트 타입으로 "검증자" 팀원을 스폰해줘 — 제안서의 논리적 허점과 리스크를 검증한다.

조사자 2명이 먼저 작업하고, 조사가 끝나면 설계자와 집필자가 작업을 시작한다.
검증자는 초안이 완료된 후 검토한다.
팀원들이 모두 완료할 때까지 기다려줘.
```

## 태스크 흐름

```
[배경조사: 현황/선행연구] ──┐
                            ├──→ [설계자: 솔루션 설계] ──→ [집필자: 제안서 작성] ──→ [검증자: 검토]
[기술조사: 기술 분석]     ──┘                                                          │
                                                                                       └──→ [리드: 최종 정리]

※ 병렬: 배경조사 ∥ 기술조사 (서로 다른 주제 영역)
※ 직렬: 설계자 → 집필자 → 검증자 (각 단계가 앞 단계의 출력에 강하게 의존).
※ 집필자가 작성 중 추가 조사가 필요하다고 판단하면 리드가 조사자를 재투입한다.
```

## 결과물 구조 (권장)

경로는 CLAUDE.md의 "작업 디렉토리 규칙"을 따른다. `{작업디렉토리}`는 사용자가 지정한 폴더거나, 미지정 시 `output/{작업슬러그}/`이다.

```
{작업디렉토리}/
├── proposal.md          # 최종 제안서
├── research/
│   ├── background.md    # 배경 조사 결과
│   └── technical.md     # 기술 조사 결과
├── architecture.md      # 시스템 설계
└── review.md            # 검증 결과 및 반영 사항
```

## 변형

### 간단한 제안서 (팀원 3명)
조사자 1명 + 집필자 + 검증자

### 경쟁 제안서 (팀원 6명)
집필자를 2명으로 나눠 서로 다른 접근법으로 작성 후 비교
