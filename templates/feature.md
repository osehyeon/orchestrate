# 코드 기능 개발 템플릿

## 사용법

Claude Code에서 아래 프롬프트의 `{{변수}}`를 실제 값으로 바꿔서 입력한다.

## 프롬프트

```
다음 기능을 개발하기 위한 에이전트 팀을 만들어줘.

**기능 목표**: {{기능 설명}}
**대상 프로젝트**: {{프로젝트 경로 또는 설명}}
**기술 스택**: {{사용하는 언어/프레임워크}}
**제약 조건**: {{일정, 성능 요구사항 등}}

팀 구성:
1. architect 에이전트 타입으로 "설계자" 팀원을 스폰해줘 — 기능의 아키텍처를 설계하고, 구현 계획을 수립한다. plan approval을 요구해줘. 리드가 자율적으로 계획의 타당성을 판단하여 승인/거부한다.
2. researcher 에이전트 타입으로 "조사자" 팀원을 스폰해줘 — 기존 코드베이스를 분석하고, 영향 받는 파일과 패턴을 파악한다.
3. implementer 에이전트 타입으로 "구현자A" 팀원을 스폰해줘 — 핵심 로직을 구현한다. 설계자의 계획이 승인된 후 시작한다.
4. implementer 에이전트 타입으로 "구현자B" 팀원을 스폰해줘 — 테스트 코드를 작성한다. 설계자의 계획이 승인된 후 시작한다.
5. reviewer 에이전트 타입으로 "검증자" 팀원을 스폰해줘 — 구현이 완료되면 코드 리뷰를 수행한다.

팀원들이 모두 완료할 때까지 기다려줘.
```

## 태스크 흐름

```
[설계자: 아키텍처 설계] ──┐
                          ├──→ [구현자A: 핵심 로직] ──┐
[조사자: 코드베이스 분석] ──┘    [구현자B: 테스트]    ──┤──→ [검증자: 코드 리뷰]
                                                       │
                                                       └──→ [리드: 통합 및 정리]
```

## 변형

### 소규모 기능 (팀원 3명)
설계자 + 구현자 + 검증자로 축소 가능

### 대규모 기능 (팀원 6-7명)
구현자를 프론트엔드/백엔드/DB로 분리 가능
