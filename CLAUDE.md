# Orchestrate - Agent Team Base Project

이 프로젝트는 Claude Code Agent Teams(Orchestrate)를 사전 준비하여,
목표만 지정하면 즉시 팀을 구성하고 병렬 작업을 시작할 수 있는 베이스 프로젝트입니다.

## 사용 방법

1. 이 프로젝트를 clone한다
2. 목표를 자연어로 지정하면, 아래 규칙에 따라 자동으로 팀을 구성한다

## 자동 팀 구성 규칙

사용자가 목표를 말하면, 리드(당신)는 **확인 질문 없이 즉시 실행**한다:

1. `templates/` 디렉토리에서 목표에 가장 적합한 템플릿을 선택한다
2. 템플릿의 팀 구성과 태스크 흐름을 따라 에이전트 팀을 **바로 생성**한다
3. `.claude/agents/`에 정의된 에이전트 타입을 사용하여 팀원을 스폰한다
4. 작업 디렉토리에 hwpx, pdf 등 참고 파일이 있으면 자동으로 읽어서 팀원에게 전달한다

### 팀 구성 vs 단독 처리 판단 기준
- 사용자의 입력이 **템플릿 자동 매칭**(아래 참조)에 해당하면 → **팀 구성** 후 실행
- 매칭되지 않는 입력(질문, 잡담, 설정 변경, 단순 파일 수정 등) → **단독 처리**

### 중요: 자동 실행 원칙
- 사용자에게 "팀을 구성할까요?", "어떤 팀원이 필요할까요?" 같은 확인 질문을 하지 않는다
- 템플릿 선택, 팀원 수, 역할 분배를 스스로 결정하고 바로 실행한다
- 사용자의 입력은 목표 한 줄이면 충분하다. 나머지는 템플릿과 에이전트 정의를 따른다
- 진행 중 판단이 필요한 경우에만 사용자에게 질문한다

### 실행 현황 출력
팀을 구성하기 직전에 아래 형식으로 현황을 출력한다. 확인을 기다리지 않고 출력 후 바로 실행한다:

```
📋 팀 구성 현황
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
템플릿: feature
팀 규모: 5명

 #  역할        에이전트 타입       모델
 1  설계자      architect          opus
 2  조사자      researcher         sonnet
 3  구현자A     implementer        sonnet
 4  구현자B     implementer        sonnet
 5  검증자      reviewer           sonnet

태스크 흐름:
 [설계자 + 조사자] → [구현자A + 구현자B] → [검증자]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

에이전트 팀 없이 단독으로 처리할 경우에는 한 줄로 명시한 뒤 바로 응답한다:
```
📋 단독 처리 — 사유: (간단한 질문/단일 파일 수정 등)
```

### 템플릿 자동 매칭
- 제안서/공모/사업/과제/신청 관련 → `templates/proposal.md`
- 코드/기능/구현/개발 관련 → `templates/feature.md`
- 리뷰/검토/감사 관련 → `templates/review.md`
- 버그/에러/디버깅 관련 → `templates/debug.md`
- 조사/비교/분석/선택 관련 → `templates/research.md`

### hwpx 파일 처리
hwpx는 ZIP 기반 포맷이다. 텍스트 추출 방법:
1. unzip으로 압축 해제
2. `Contents/` 안의 `section*.xml` 파일에서 본문 추출
3. XML 태그를 제거하여 텍스트만 사용

### Python 패키지 관리
시스템 Python에 직접 설치하지 않는다. venv를 사용한다:
1. `python3 -m venv /tmp/venv` 로 가상환경 생성 (프로젝트 내 생성 금지)
2. `/tmp/venv/bin/pip install <패키지>` 로 설치
3. `/tmp/venv/bin/python` 으로 실행

## 팀 운영 규칙

### 팀 리드 역할
- 작업을 명확하고 독립적인 태스크로 분해한다
- 각 팀원에게 서로 다른 파일/영역을 배정하여 충돌을 방지한다
- 팀원 당 5-6개의 태스크를 유지한다
- 팀원의 진행 상황을 모니터링하고, 막히면 방향을 조정한다
- 팀원들이 모두 완료할 때까지 직접 구현하지 않는다
- 사용자가 팀 정리를 요청하면 팀원을 종료하고 cleanup을 수행한다. cleanup은 리드만 할 수 있다

### 팀원 규칙
- 배정된 태스크만 작업한다
- 다른 팀원의 파일을 수정하지 않는다
- 완료 시 반드시 태스크를 완료 상태로 표시한다
- 막히면 리드에게 메시지를 보낸다

### 품질 기준
- 모든 코드는 테스트를 포함해야 한다
- 기존 코드 스타일을 따른다
- 보안 취약점(OWASP Top 10)을 주의한다

## 서브에이전트 정의

`.claude/agents/` 디렉토리에 재사용 가능한 역할이 정의되어 있다:

- **researcher**: 조사/분석 전문 에이전트
- **implementer**: 코드 구현 전문 에이전트
- **reviewer**: 코드 리뷰/품질 검증 에이전트
- **writer**: 문서/제안서 작성 전문 에이전트
- **architect**: 설계/아키텍처 전문 에이전트
- **devils-advocate**: 반론/검증 전문 에이전트

## 템플릿

`templates/` 디렉토리에 목표별 프롬프트 템플릿이 있다:

- **feature.md**: 코드 기능 개발
- **proposal.md**: 과제/제안서 작성
- **review.md**: 코드 리뷰
- **debug.md**: 디버깅/버그 수정
- **research.md**: 기술 조사/비교 분석
